Proyecto 12 – Control de Accesos por QR a Restaurantes :restaurant: :qr_code::book: Descripción GeneralEste proyecto consiste en el desarrollo de una aplicación web integral diseñada para automatizar el control de aforo en tiempo real en el sector de la restauración. El sistema permite a los establecimientos generar códigos QR únicos que los clientes escanean al entrar y salir, garantizando una gestión del flujo de personas precisa y transaccional.:warning: El ProblemaLa gestión actual del aforo suele depender de actualizaciones manuales por parte del personal, lo que genera información desfasada y ambigüedad para el cliente, especialmente en horas punta.:bulb: La SoluciónAutomatizar el registro de entradas y salidas mediante tecnología QR para eliminar el error humano y ofrecer disponibilidad 100% real a los usuarios.:target: Objetivos del SistemaControl de Aforo: Implementar un motor transaccional para el registro de entradas y salidas.Generación de QR: Crear un sistema de validación de códigos únicos por restaurante.Métricas de Afluencia: Proporcionar al hostelero datos sobre duración media de visitas e historial de ocupación.Visibilidad para el Cliente: Ofrecer un listado de restaurantes filtrables por disponibilidad real y tipo de comida.:gear: Metodología y HerramientasEl proyecto se desarrolla siguiendo la metodología ágil SCRUMBAN, gestionando el ciclo de vida del software mediante GitHub Projects y Issues.HerramientaUsoGit / GitHubControl de versiones y gestión de tareas.MarkdownDocumentación técnica y memorias.UWE (UML-Based Web Engineering)Modelado de requisitos y arquitectura.OpenCode / EngramAsistentes de IA para acelerar el desarrollo.:group: Organización del EquipoAl tratarse de un desarrollo individual, se asumen todos los roles definidos en la metodología de la asignatura para asegurar el cumplimiento del ciclo de vida del software:Scrum Master: Gestión del backlog y tableros Kanban.Desarrollador Full-Stack: Implementación de Front-end, Back-end y APIs.Gestor de BD: Diseño y mantenimiento del modelo de datos.Analista de Requisitos: Modelado UWE y validación.:chart_with_upwards_trend: Estado del Proyecto[!NOTE]
